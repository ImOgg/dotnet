# Markdown 筆記整理 Skill

> 專為整理技術學習筆記設計，適用於本專案 `docs/` 目錄下的 ASP.NET Core 相關文件。

## 整理原則

- **保留內容**：只調整格式，不改變原有技術內容的意涵
- **結構清晰**：使用標題層級建立清楚的閱讀脈絡
- **中文優先**：本專案筆記使用繁體中文，確保語句通順自然
- **一致性**：同類型內容使用相同的格式模式
- **精簡扼要**：刪除廢話和重複敘述，每句話都要有資訊量；能用列表就不用長段落，能用一句話說清楚就不寫三句
- **無情緒價值**：移除「這個概念非常重要！」、「學會這個你就厲害了」之類的情緒填充語，只保留技術內容

## 執行流程

整理 markdown 筆記時，**依序**完成以下步驟：

1. **讀取檔案** — 使用 Read 工具讀取目標 markdown 檔案
2. **分析現況** — 識別結構問題、格式不一致、缺少的元素
3. **整理內容** — 依照格式規範和結構模式進行調整
4. **輸出結果** — 使用 Edit 或 Write 工具儲存，並說明主要修改項目

## 快速參考

### 標準筆記開頭

```markdown
# 主題標題

> 一句話摘要：說明這篇筆記涵蓋的核心內容。

## 前置知識
- 需要了解...

## 目錄（選用，文章較長時加入）
- [章節一](#章節一)
- [章節二](#章節二)
```

### 代碼塊規範

````markdown
```bash
# 指令說明
dotnet ef migrations add MigrationName
```

```csharp
// 代碼說明
public class Example { }
```
````

### 重點標示

```markdown
> **注意**：需要特別留意的事項。

> **提示**：有用的小技巧。

> **警告**：可能導致問題的操作。
```

### 常見整理模式

| 原始寫法 | 整理後 |
|----------|--------|
| `##沒有空格` | `## 有空格` |
| 混用 `*` 和 `-` 列表 | 統一使用 `-` |
| 代碼無語言標注 ` ``` ` | 加上語言 ` ```csharp ` |
| 段落間無空行 | 段落間保留一個空行 |
| 中英文無空格 `ASP.NETCore` | 中英文間加空格 `ASP.NET Core` |
